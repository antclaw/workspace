# Contributing to Awesome Tools

感谢你考虑为 Awesome Tools 贡献内容！

## 如何贡献

### 1. 选择一个分类

确保你的工具符合以下分类之一：
- Productivity Tools
- Development Tools
- Design Tools
- File Management
- Communication Tools
- Browser Tools
- Note Taking
- Calendar & Scheduling
- Task Management
- Time Tracking
- Code Editors
- Git Tools
- Docker Tools
- Cloud Tools
- API Tools
- Testing Tools
- Security Tools
- Data Analysis
- Database Tools
- DevOps Tools
- AI Tools
- Browser Extensions
- Mobile Apps

### 2. 准备内容

- **标题**: 工具的清晰标题
- **描述**: 简短描述（1-2 句话）
- **链接**: GitHub 或官方网站链接
- **分类**: 指定正确的分类
- **特点**: 列出 3-5 个主要特点

### 3. 提交 PR

```bash
# Fork 本仓库
git fork shadow/awesome-tools

# 创建分支
git checkout -b feature/add-awesome-tool

# 添加你的工具
echo "- **[标题](链接)** - 简短描述\n  - 特点 1\n  - 特点 2\n  - 特点 3" >> README.md

# 提交
git commit -m "Add [标题]"

# 推送
git push origin feature/add-awesome-tool

# 创建 PR
```

### 4. PR 格式

请按照以下格式组织你的 PR：

```markdown
## 新增工具：[标题]

- **[标题](链接)** - 简短描述
  - 特点 1
  - 特点 2
  - 特点 3
  - 特点 4
  - 特点 5

**分类**: [分类名称]

**链接**: [GitHub/Website]

**特点**:
- [特点 1]
- [特点 2]
- [特点 3]
```

## 贡献指南

### 内容质量

- ✅ 确保链接有效
- ✅ 描述准确简洁
- ✅ 特点有价值
- ✅ 分类正确
- ✅ 工具最新

### 代码风格

- 使用 Markdown 列表
- 保持一致的格式
- 使用粗体强调标题
- 使用项目符号列出特点
- 使用引用块列出主要特点

### 检查清单

在提交 PR 前，请检查：
- [ ] 链接有效
- [ ] 描述清晰
- [ ] 特点有价值
- [ ] 分类正确
- [ ] 格式一致
- [ ] 工具是最新版本

## 社区规范

- 尊重所有贡献者
- 保持专业和友好
- 接受反馈和建议
- 遵循 PR 审核流程
